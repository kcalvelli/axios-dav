# Agent Instructions for axios-dav

## Your Role

You are an AI assistant helping develop axios-dav, a declarative NixOS module for CalDAV/CardDAV synchronization with MCP integration.

## Mandatory Workflow

### Before ANY Code Changes

1. **Check for existing proposal** in `openspec/changes/`
2. **If none exists**, create one:
   ```
   openspec/changes/[descriptive-name]/
   ├── proposal.md    # What and why
   └── tasks.md       # Step-by-step implementation plan
   ```
3. **Get user approval** before implementing

### During Implementation

1. Follow `tasks.md` step by step
2. Check off completed tasks
3. Update specs if behavior changes
4. Commit frequently with clear messages

### After Implementation

1. Verify all tasks complete
2. Update `openspec/specs/` with final state
3. Move proposal to `openspec/changes/archive/`
4. Push all changes

## Project-Specific Guidelines

### NixOS Module Patterns

Follow standard NixOS module structure:
```nix
{ config, lib, pkgs, ... }:
let
  cfg = config.services.axios-dav;
in
{
  options.services.axios-dav = {
    enable = lib.mkEnableOption "axios-dav calendar and contacts sync";
    # ... more options
  };

  config = lib.mkIf cfg.enable {
    # Implementation
  };
}
```

### Config Generation

Generate config files using Nix:
```nix
home.file.".config/vdirsyncer/config".text = ''
  # Generated by axios-dav - do not edit manually
  ${generatedConfig}
'';
```

### Secrets Handling

Always use file paths for secrets:
```nix
tokenFile = lib.mkOption {
  type = lib.types.path;
  description = "Path to OAuth token file (e.g., from agenix)";
};
```

Never:
- Hardcode credentials
- Use environment variables for secrets
- Store secrets in Nix store

### MCP Server Development

The mcp-dav server should:
- Be a Python package in `pkgs/mcp-dav/`
- Use the official `mcp` Python SDK
- Provide tools for both calendar and contacts
- Read from local vdirsyncer-managed files
- Write by creating/modifying local files + triggering sync

### Testing

Before marking complete:
- `nix flake check` passes
- Module evaluates without errors
- Config generation produces valid vdirsyncer config
- MCP server starts and responds to tool calls

- Explain architectural decisions

## When Stuck

1. Check `openspec/discovery/unknowns.md` - is this a known gap?
2. If not, add it and ask the user for guidance
3. Don't guess at requirements - ask
